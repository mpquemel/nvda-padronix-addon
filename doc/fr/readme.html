1. À propos de Padronix

Padronix est un module complémentaire NVDA qui structure et formate automatiquement les articles académiques à l'aide de l'intelligence artificielle.

Caractéristiques principales:
  • Structuration automatique des articles via IA
  • 7 normes académiques: ABNT, APA, IEEE, Chicago, Harvard, MLA, Vancouver
  • 4 fournisseurs d'IA: Google Gemini, OpenAI, OpenRouter, Ollama Local
  • Interface 100% accessible avec NVDA
  • Retour audio tout au long du traitement
  • Support de 7 langues

2. Installation

2.1 Prérequis

  • NVDA: Version 2024.1 ou ultérieure
  • Windows: 10 ou 11
  • Python: 3.11+ (inclus avec NVDA)

2.2 Étapes d'installation

  1. Téléchargez le fichier padronix-2026.1.0.nvda-addon
  2. Avec NVDA ouvert, appuyez sur NVDA+N pour ouvrir le menu
  3. Naviguez vers Outils → Gérer les modules complémentaires
  4. Appuyez sur I pour Installer
  5. Sélectionnez le fichier .nvda-addon téléchargé
  6. Confirmez l'installation
  7. Redémarrez NVDA lorsque demandé
✅ Vérification: Après le redémarrage, appuyez sur NVDA+Ctrl+Shift+F. Vous devriez entendre: "Padronix ouvert".

3. Configuration des fournisseurs d'IA

3.1 Google Gemini (Recommandé pour le contexte long)

Comment obtenir la clé API Google:

  1. Visitez Google AI Studio
  2. Connectez-vous avec votre compte Google
  3. Cliquez sur "Create API Key"
  4. Sélectionnez ou créez un projet
  5. Copiez la clé générée (commence par AIza...)
  6. Conservez-la en lieu sûr!
⚠️ Limite gratuite: L'API Google Gemini offre 60 requêtes par minute gratuitement.

Modèles disponibles:

  • gemini-pro - Modèle standard (rapide)
  • gemini-1.5-pro - Contexte jusqu'à 1 million de tokens
  • gemini-1.5-flash - Plus rapide, moins précis

3.2 OpenAI (GPT-4, GPT-3.5)

Comment obtenir la clé API OpenAI:

  1. Visitez OpenAI Platform
  2. Connectez-vous ou créez un compte
  3. Cliquez sur "+ Create new secret key"
  4. Nommez la clé (ex: "Padronix")
  5. Copiez immédiatement (commence par sk-proj-...)
  6. Attention: La clé n'apparaît qu'une seule fois!
💰 Coûts: OpenAI facture par token traité. GPT-4 est plus cher que GPT-3.5.

3.3 OpenRouter (Multiples modèles)

Comment obtenir la clé API OpenRouter:

  1. Visitez OpenRouter Keys
  2. Connectez-vous (Google, GitHub ou email)
  3. Cliquez sur "Create Key"
  4. Nommez-la (ex: "Padronix")
  5. Copiez la clé (commence par sk-or-...)
Avantage OpenRouter: Un seul crédit donne accès à des dizaines de modèles: Claude-3, Llama, Mistral, Gemma, et bien d'autres.

3.4 Ollama Local (100% offline et gratuit)

Étape 1: Télécharger et installer Ollama

  1. Visitez ollama.ai/download
  2. Cliquez sur "Download for Windows"
  3. Exécutez OllamaSetup.exe
  4. Suivez les instructions d'installation
  5. Ollama démarre automatiquement après l'installation

Étape 2: Télécharger un modèle

Ouvrez l'Invite de commandes ou PowerShell et tapez:

ollama pull llama3

ollama run llama3 "Bonjour, comment allez-vous?"
Si vous recevez une réponse en français, ça fonctionne!

4. Comment utiliser

4.1 Ouvrir Padronix

Appuyez sur: NVDA+Ctrl+Shift+F

Vous entendrez: "Padronix ouvert"

4.2 Flux de travail complet

Étape 1: Charger l'article

  1. Avec la fenêtre Padronix ouverte, appuyez sur Alt+C
  2. Naviguez jusqu'à votre fichier TXT
  3. Appuyez sur Enter
  4. Vous entendrez: "TXT chargé: [nombre] caractères"

Étape 2: Sélectionner la norme

  1. Appuyez sur Tab jusqu'à entendre "Norme académique"
  2. Utilisez Flèche haut/bas pour choisir: ABNT, APA, IEEE, etc.

Étape 3: Générer le DOCX

  1. Appuyez sur Alt+G
  2. Entendez le retour: "Structuration de l'article avec l'IA..."
  3. Attendez (30-90 secondes)
  4. Son de succès (accord de piano)
  5. Vous entendrez: "Document Padronix généré à: [chemin]"
🎵 Retour sonore: Pendant le traitement, vous entendrez un bip doux toutes les 3 secondes. Cela indique que Padronix fonctionne.

7. Dépannage

7.1 "Erreur lors de l'ouverture de Padronix"

Cause: Dépendances non installées (python-docx, requests)

Solution:

pip install python-docx requests

7.2 "Ollama offline"

Cause: Ollama ne fonctionne pas

Solution:

ollama serve

7.3 "Délai d'attente dépassé"

Cause: L'IA a pris plus de 60 secondes

Solution:

  • Utilisez des modèles plus petits (llama3 au lieu de llama3:70b)
  • Divisez les articles très longs (>50 pages)
  • Vérifiez votre connexion internet
