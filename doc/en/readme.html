1. About Padronix

Padronix is an NVDA add-on that automatically structures and formats academic papers using Artificial Intelligence.

Key Features:
  • Automatic article structuring via AI
  • 7 academic standards: ABNT, APA, IEEE, Chicago, Harvard, MLA, Vancouver
  • 4 AI providers: Google Gemini, OpenAI, OpenRouter, Ollama Local
  • 100% accessible interface with NVDA
  • Auditory feedback throughout processing
  • 7 languages supported

2. Installation

2.1 Prerequisites

  • NVDA: Version 2024.1 or later
  • Windows: 10 or 11
  • Python: 3.11+ (included with NVDA)

2.2 Installation Steps

  1. Download padronix-2026.1.0.nvda-addon
  2. With NVDA running, press NVDA+N to open menu
  3. Navigate to Tools → Manage Add-ons
  4. Press I for Install
  5. Select the downloaded .nvda-addon file
  6. Confirm installation
  7. Restart NVDA when prompted (press NVDA+Q, then Enter)
✅ Verification: After restart, press NVDA+Ctrl+Shift+F. You should hear: "Padronix opened".

3. AI Provider Setup

3.1 Google Gemini (Recommended for long context)

How to get Google API Key:

  1. Visit Google AI Studio
  2. Sign in with your Google account
  3. Click "Create API Key"
  4. Select or create a project
  5. Click "Create API Key in existing project"
  6. Copy the generated key (starts with AIza...)
  7. Store it securely!
⚠️ Free limit: Google Gemini API offers 60 requests per minute free. For intensive academic use, consider the paid plan.

Available models:

  • gemini-pro - Standard model (fast)
  • gemini-1.5-pro - Up to 1M token context
  • gemini-1.5-flash - Faster, less accurate

3.2 OpenAI (GPT-4, GPT-3.5)

How to get OpenAI API Key:

  1. Visit OpenAI Platform
  2. Sign in or create account
  3. Click "+ Create new secret key"
  4. Name the key (e.g., "Padronix")
  5. Copy immediately (starts with sk-proj-...)
  6. Warning: Key only shows once!
💰 Costs: OpenAI charges per token. GPT-4 is more expensive than GPT-3.5. Monitor usage at Usage Dashboard.

3.3 OpenRouter (Multiple models)

How to get OpenRouter API Key:

  1. Visit OpenRouter Keys
  2. Sign in (Google, GitHub, or email)
  3. Click "Create Key"
  4. Name it (e.g., "Padronix")
  5. Copy the key (starts with sk-or-...)
OpenRouter advantage: Single credit gives access to dozens of models: Claude-3, Llama, Mistral, Gemma, and many more.

3.4 Ollama Local (100% offline & free)

Step 1: Download and install Ollama

  1. Visit ollama.ai/download
  2. Click "Download for Windows"
  3. Run OllamaSetup.exe
  4. Follow installation prompts (default is fine)
  5. Ollama starts automatically after install

Step 2: Download a model

Open Command Prompt or PowerShell and type:

ollama pull llama3

4. How to Use

4.1 Open Padronix

Press: NVDA+Ctrl+Shift+F

You'll hear: "Padronix opened"

4.2 Complete Workflow

Step 1: Load article

  1. With Padronix window open, press Alt+C
  2. Navigate to your TXT file
  3. Press Enter
  4. You'll hear: "TXT loaded: [number] characters"

Step 2: Select standard

  1. Press Tab until you hear "Academic Standard"
  2. Use Up/Down Arrow to choose: ABNT, APA, IEEE, etc.

Step 3: Generate DOCX

  1. Press Alt+G
  2. Hear feedback: "Structuring article with AI..."
  3. Wait (30-90 seconds)
  4. Success sound (piano chord)
  5. You'll hear: "Padronix document generated at: [path]"

7. Troubleshooting

7.1 "Error opening Padronix"

Cause: Missing dependencies (python-docx, requests)

Solution:

pip install python-docx requests

7.2 "Ollama offline"

Cause: Ollama not running

Solution:

ollama serve

7.3 "Request timeout"

Cause: AI took more than 60 seconds

Solution:

  • Use smaller models (llama3 instead of llama3:70b)
  • Split very long articles (>50 pages)
  • Check internet connection
