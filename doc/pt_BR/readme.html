1. Sobre o Padronix

O Padronix é um complemento (add-on) para o NVDA que estrutura e formata artigos acadêmicos automaticamente usando Inteligência Artificial.

Recursos principais:
  • Estruturação automática de artigos via IA
  • 7 padrões acadêmicos: ABNT, APA, IEEE, Chicago, Harvard, MLA, Vancouver
  • 4 provedores de IA: Google Gemini, OpenAI, OpenRouter, Ollama Local
  • Interface 100% acessível com NVDA
  • Feedback auditivo durante todo o processamento
  • Suporte a 7 idiomas

2. Instalação

2.1 Pré-requisitos

  • NVDA: Versão 2024.1 ou superior
  • Windows: 10 ou 11
  • Python: 3.11+ (já incluso no NVDA)

2.2 Passos para Instalar

  1. Baixe o arquivo padronix-2026.1.0.nvda-addon
  2. Com o NVDA aberto, pressione NVDA+N para abrir o menu
  3. Navegue até Ferramentas → Gerenciar Complementos
  4. Pressione I para Instalar
  5. Selecione o arquivo .nvda-addon baixado
  6. Confirme a instalação
  7. Reinicie o NVDA quando solicitado (pressione NVDA+Q, depois Enter)
✅ Verificação: Após reiniciar, pressione NVDA+Ctrl+Shift+F. Você deve ouvir: "Padronix aberto".

3. Configuração dos Provedores de IA

O Padronix suporta 4 provedores de IA. Escolha um ou mais conforme sua necessidade:

3.1 Google Gemini (Recomendado para contexto longo)

Como obter a chave API do Google:

  1. Acesse Google AI Studio
  2. Faça login com sua conta Google
  3. Clique em "Create API Key" (Criar chave API)
  4. Selecione um projeto ou crie um novo
  5. Clique em "Create API Key in existing project"
  6. Copie a chave gerada (inicia com AIza...)
  7. Guarde em local seguro!
⚠️ Limite gratuito: A API do Google Gemini oferece 60 requisições por minuto gratuitamente. Para uso acadêmico intensivo, considere o plano pago.

Modelos disponíveis:

  • gemini-pro - Modelo padrão (rápido)
  • gemini-1.5-pro - Contexto de até 1 milhão de tokens
  • gemini-1.5-flash - Mais rápido, menos preciso

3.2 OpenAI (GPT-4, GPT-3.5)

Como obter a chave API da OpenAI:

  1. Acesse OpenAI Platform
  2. Faça login ou crie uma conta
  3. Clique em "+ Create new secret key"
  4. Dê um nome para a chave (ex: "Padronix")
  5. Copie a chave imediatamente (inicia com sk-proj-...)
  6. Atenção: A chave só aparece uma vez!
💰 Custos: A OpenAI cobra por token processado. GPT-4 é mais caro que GPT-3.5. Monitore seu uso em Usage Dashboard.

Modelos disponíveis:

  • gpt-4-turbo - Mais inteligente, mais caro
  • gpt-4 - Qualidade consistente
  • gpt-3.5-turbo - Rápido e econômico

3.3 OpenRouter (Múltiplos modelos)

Como obter a chave API do OpenRouter:

  1. Acesse OpenRouter Keys
  2. Faça login (Google, GitHub ou email)
  3. Clique em "Create Key"
  4. Dê um nome (ex: "Padronix")
  5. Copie a chave (inicia com sk-or-...)
Vantagem do OpenRouter: Um único crédito dá acesso a dezenas de modelos: Claude-3, Llama, Mistral, Gemma, e muitos outros. Ideal para testar diferentes IAs.

Modelos populares no OpenRouter:

  • anthropic/claude-3-sonnet - Excelente para texto
  • meta-llama/llama-3-70b-instruct - Open source de alta qualidade
  • google/gemini-pro-1.5 - Contexto longo
  • mistralai/mistral-large - Europeu, multilíngue

3.4 Ollama Local (100% offline e gratuito)

Passo 1: Baixar e instalar o Ollama

  1. Acesse ollama.ai/download
  2. Clique em "Download for Windows"
  3. Execute o instalador OllamaSetup.exe
  4. Siga as instruções de instalação (padrão é suficiente)
  5. Ao final, o Ollama inicia automaticamente

Passo 2: Baixar um modelo

Abra o Prompt de Comando ou PowerShell e digite:

ollama pull llama3

Este comando baixa o modelo Llama 3 (8 bilhões de parâmetros). Outros modelos:

Modelos recomendados para Ollama
Modelo Tamanho RAM necessária Uso recomendado
llama3 8B 8 GB Uso geral (recomendado)
llama3:70b 70B 32 GB Qualidade máxima
mistral 7B 8 GB Rápido e eficiente
codellama 7B 8 GB Código e técnica
gemma:7b 7B 8 GB Google, leve

Passo 3: Verificar se o Ollama está rodando

ollama list

Deve mostrar os modelos instalados. Se não mostrar, inicie o servidor:

ollama serve
✅ Teste rápido: No PowerShell, digite: 
ollama run llama3 "Olá, como você está?"
Se receber uma resposta em português, está funcionando!

Ollama na Nuvem (alternativa para PCs fracos)

Se seu computador não tem RAM suficiente, use serviços de nuvem que rodam Ollama:

  • Google Cloud Vertex AI
  • Amazon SageMaker
  • Hugging Face Spaces

4. Como Usar

4.1 Abrir o Padronix

Pressione: NVDA+Ctrl+Shift+F

Você ouvirá: "Padronix aberto"

4.2 Fluxo de Trabalho Completo

Etapa 1: Carregar o artigo

  1. Com a janela do Padronix aberta, pressione Alt+C
  2. Navegue até seu arquivo TXT
  3. Pressione Enter
  4. Você ouvirá: "TXT carregado: [número] caracteres"

Etapa 2: Configurar o padrão

  1. Pressione Tab até ouvir "Padrão Acadêmico"
  2. Use Seta para cima/baixo para escolher: ABNT, APA, IEEE, etc.

Etapa 3: Gerar o DOCX

  1. Pressione Alt+G
  2. Ouça o feedback: "Estruturando artigo com IA..."
  3. Aguarde (pode levar 30-90 segundos)
  4. Som de sucesso (acorde de piano)
  5. Você ouvirá: "Documento Padronix gerado em: [caminho]"
🎵 Feedback sonoro: Durante o processamento, você ouvirá um beep suave a cada 3 segundos. Isso indica que o Padronix está trabalhando e não travou.

7. Solução de Problemas

7.1 "Erro ao abrir Padronix"

Causa provável: Dependências não instaladas (python-docx, requests)

Solução:

pip install python-docx requests

7.2 "Ollama offline"

Causa provável: Ollama não está rodando

Solução:

ollama serve

Ou reinicie o computador.

7.3 "Timeout na requisição"

Causa provável: IA demorou mais de 60 segundos

Solução:

  • Use modelos menores (llama3 em vez de llama3:70b)
  • Divida artigos muito longos (>50 páginas)
  • Verifique sua conexão com a internet

7.4 "JSON inválido da IA"

Causa provável: Modelo de IA não seguiu o formato solicitado

Solução:

  • Tente outro modelo (Gemini ou GPT-4 são mais consistentes)
  • Reduza o tamanho do texto de entrada
  • Verifique se o texto está em formato legível
